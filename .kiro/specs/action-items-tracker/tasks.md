# Tasks: Action Items Tracker & History

## Task List

- [ ] 1. Database schema and migrations
  - [ ] 1.1 Write and apply Supabase migration: create `action_items` table with columns (`id`, `sprint_id`, `assignee_id`, `description`, `status`, `source_card_id`, `created_at`), constraints (`status CHECK IN ('todo','in_progress','done')`, `char_length(description) BETWEEN 1 AND 500`), foreign keys (`sprint_id → sprints`, `assignee_id → auth.users ON DELETE RESTRICT`, `source_card_id → retro_cards ON DELETE SET NULL`), and RLS policies (SELECT/INSERT/UPDATE/DELETE scoped to team members via `sprint_id → sprints → team_members` join)
  - [ ] 1.2 Create `types/action-items.ts` with `ActionItemStatus`, `ActionItem`, `ActionItemWithAssignee`, `ActionItemError`, `ActionItemErrorCode`, `CreateActionItemData`, `UpdateActionItemData`, `ActionItemResult`, `DeleteResult`, `UnresolvedCalloutData` types

- [ ] 2. Validator (`lib/action-items/validators.ts`)
  - [ ] 2.1 Implement `validateDescription` — non-empty, at least one non-whitespace character, at most 500 characters
  - [ ] 2.2 Implement `validateAssigneeId` — non-empty, valid UUID format
  - [ ] 2.3 Implement `validateStatusTransition(current, next)` — accepts only the three valid transitions (`todo→in_progress`, `in_progress→done`, `done→in_progress`); returns `valid: false` with descriptive message for all other pairs

- [ ] 3. ActionItem_Service (`lib/action-items/service.ts`)
  - [ ] 3.1 Implement `getActionItemsForSprint(sprintId)` — fetch all `action_items` records for the given `sprint_id` using server client, ordered by `created_at` ascending; join `auth.users` to include assignee `display_name`; return empty array if none found
  - [ ] 3.2 Implement `getActionItemById(actionItemId)` — fetch single `action_items` record by `id`; return `null` if not found
  - [ ] 3.3 Implement `getUnresolvedItemsFromLastCompletedSprint(teamId)` — fetch the single most recently completed sprint for the team (highest `sprint_number` where `status = 'completed'`); fetch all `action_items` for that sprint where `status IN ('todo', 'in_progress')`; return `null` if no completed sprint exists or all items are `done`; join assignee display names
  - [ ] 3.4 Implement `createActionItem(data, requestingUserId, isFacilitator)` — return `FORBIDDEN` if `isFacilitator` is false; validate `description` and `assignee_id`; verify `assignee_id` is a member of the sprint's team; insert record with `status = 'todo'` and provided `source_card_id` (null for manual creation); return created `ActionItem` on success
  - [ ] 3.5 Implement `updateActionItem(actionItemId, data, requestingUserId, isFacilitator)` — return `FORBIDDEN` if `isFacilitator` is false; validate `description` and `assignee_id`; update `description` and `assignee_id` on the record; return updated `ActionItem` on success
  - [ ] 3.6 Implement `deleteActionItem(actionItemId, requestingUserId, isFacilitator)` — return `FORBIDDEN` if `isFacilitator` is false; delete the record; return `{ success: true }` on success
  - [ ] 3.7 Implement `transitionStatus(actionItemId, newStatus, requestingUserId)` — fetch the current `ActionItem`; return `FORBIDDEN` if `requestingUserId` does not match `assignee_id` and the user is not a facilitator (pass `isFacilitator` flag); validate the transition using `validateStatusTransition`; return `INVALID_STATUS_TRANSITION` if invalid; update `status` in DB; return updated `ActionItem` on success
  - [ ] 3.8 Implement `carryForward(actionItemId, requestingUserId, isFacilitator)` — return `FORBIDDEN` if `isFacilitator` is false; fetch the original `ActionItem`; find the active sprint for the team (`status = 'active'`); return `NO_ACTIVE_SPRINT` if none exists; insert a new `action_items` record copying `description` and `assignee_id`, with `status = 'todo'`, `source_card_id = null`, and the active sprint's `sprint_id`; do NOT modify the original record; return the new `ActionItem` on success

- [ ] 4. Route Handlers
  - [ ] 4.1 Create `app/api/teams/[teamId]/sprints/[sprintId]/action-items/route.ts` — GET handler: authenticate, verify team membership, call `getActionItemsForSprint`, return `{ data: ActionItem[] }`; POST handler: authenticate, verify team membership and sprint ownership, determine `isFacilitator`, validate body, call `createActionItem`, return `{ data: ActionItem }` on success or `{ error: ActionItemError }` with appropriate HTTP status
  - [ ] 4.2 Create `app/api/teams/[teamId]/sprints/[sprintId]/action-items/[actionItemId]/route.ts` — PATCH handler: authenticate, verify team membership, determine `isFacilitator`, validate body, call `updateActionItem`, return `{ data: ActionItem }` on success or `{ error: ActionItemError }`; DELETE handler: authenticate, verify team membership, determine `isFacilitator`, call `deleteActionItem`, return `{ data: { success: true } }` on success or `{ error: ActionItemError }`
  - [ ] 4.3 Create `app/api/teams/[teamId]/sprints/[sprintId]/action-items/[actionItemId]/status/route.ts` — PATCH handler: authenticate, verify team membership, determine `isFacilitator`, extract `newStatus` from body, call `transitionStatus`, return `{ data: ActionItem }` on success or `{ error: ActionItemError }` with appropriate HTTP status (400 for invalid transition, 403 for forbidden)
  - [ ] 4.4 Create `app/api/teams/[teamId]/sprints/[sprintId]/action-items/[actionItemId]/carry-forward/route.ts` — POST handler: authenticate, verify team membership, determine `isFacilitator`, call `carryForward`, return `{ data: ActionItem }` on success or `{ error: ActionItemError }` with appropriate HTTP status (403 for forbidden, 409 for no active sprint)
